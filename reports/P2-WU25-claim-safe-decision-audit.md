# P2-WU25 claim-safe Paper 2 decision audit

Status: decision logic complete; the repository evidence snapshot correctly
remains pending until registered real observations and their audits exist.

The executable audit selects one of five preregistered branches:

- successful event-resolved attribution;
- coastal ocean-product or preprocessing evaluation;
- physically interpretable non-detection constraints;
- a single-case short paper without cross-typhoon generalization; or
- pending evidence.

Missing evidence is never interpreted as a negative scientific result. A
failure branch is available only after the multi-event design, physical-effect
closure, direct/deformation separation, multi-source validation, sensitivity
analysis, failure log, held-out event evaluation, quiet-window false-alarm gate,
and event-SNR reporting are all complete. The single-case branch also requires a
complete physical closure, controls, sensitivity, failure log and event SNR; a
positive coefficient alone is insufficient.

The full attribution claim additionally requires a positive event-block
bootstrap interval, improvement in both RMSE and explained variance for every
fully held-out event, and at least three of the four locked novelty requirements.
Event SNR is reported for every event but is not silently promoted to a new
confirmatory success criterion. Its frozen interpretation threshold only
distinguishes a low-sensitivity null branch from failure despite an
interpretable predicted signal.

Run the current evidence audit with:

```bash
make audit-paper2-decision
```

The checked-in configuration is explicitly labelled as a placeholder and must
only be replaced by values linked to registered real-data artifacts. Data-license
review controls manuscript release separately from the scientific branch.
